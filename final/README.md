## 📁 Directory Structure

This folder contains the complete artifacts for the Algorithm Engineering course project.

### 1. `algos/`
Contains the core algorithmic logic implemented for the project.
*   **`python/`**: Includes `baswana_sen.py`, `advanced_spanner.py` (HAS Optimization), and `greedy_spanner.py` (baseline).
*   **`cpp/`**: Performance-optimized C++ implementations used for high-scale benchmarking.

### 2. `experiments/`
Contains the engineering infrastructure used to validate and benchmark the algorithms.
*   **`benchmarks/`**: Scripts for scaling, stretch analysis, and fault tolerance studies.
*   **`data_loaders/`**: Utilities for handling SNAP (Social/Road) and synthetic graph datasets.
*   **`tests/`**: A suite of 25 unit tests ensuring mathematical correctness and stretch invariants.

### 3. `results/`
Contains the empirical evidence generated by our study.
*   **`plots/`**: 9 high-quality figures (PNG/PDF) illustrating our findings (embedded in the main report).
*   Raw CSV/JSON data files used for generating the visualizations.

### 4. `report/report.md` (Main Submission)
The exhaustive final project report (1,100+ lines). This is the primary submission document containing the abstract, history, theory, methodology, analysis, and conclusions.

## 📝 Key Contribution: HAS Optimization
We have introduced the **Hybrid Adaptive Spanner (HAS)** which combines degree-weighted sampling and greedy pruning. Our experiments show a **17–50% reduction in edge density** compared to standard Baswana-Sen while strictly maintaining the theoretical $(2k-1)$ stretch guarantee.
